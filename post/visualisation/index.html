Visualisation

When I was young, I didn’t give a shi* about presentation.

I recall a school project, where the evaluation matrix specifically said presentation would make up 50% of the score.

I think I only presented numbers and text.

And gave up 50% of the score.

Fast forward 20 years, and I’m now wiser to a truth that I had no inkling of as a child: humans are visual animals.

As my blog is about data science, I think it’s fitting to start one of the earlier posts about visualisation, because future posts are going to rely on this.

As many data people will relate to, I work in an environment where most data visualisations are still created using Microsoft Excel or PowerPoint.

They’re functional. They get the job done. But they lack life.

When all charts start to look the same, insights start to blur.

I don’t promote a completely different look to each visualisation; that would be too chaotic. But I think analyst should develop their own style.

In developing this style, I some tips from Stephen Few Show Me the Numbers:

  1. Reduce the non-data ink
  2. Enhancing the data ink

This means to reduce the ink that does not pertain to the data being presented, then to emphasise the most important elements that do pertain to the data.

I’ll go through this using examples below.

Of course, the art of data visualisation cannot be explained by a blog post. I’m only putting down some of my most common processes into this post for future reference.

In order to go through some examples, I had to pick a data set. My friends and I have been talking about retirement, lately. So I’ve picked a data set relating to this topic: Life Expectancy.

R is my go to, so any code you see below can be followed in RStudio.

Below, I’ll go through how I create various types of charts:

  • Bar chart
  • Line chart
  • Scatter plot

Libraries

A few libraries were used through this analysis.

If you want to follow through in R, make sure to have these installed.

library(WDI)     # World bank data API
library(dplyr)   # data manipulation
library(tidyr)   # data manipulation
library(ggplot2) # visualisation

Download data from World Bank

The data used is available from the World Bank.

The R package WDI helps to search and download data from the World Bank by providing the World Bank’s indicator for the data set.

We can do a search as follows to find a dataset that we want, :

WDIsearch(string = "life expectancy at birth")
##      indicator           name                                      
## [1,] "SP.DYN.LE00.MA.IN" "Life expectancy at birth, male (years)"  
## [2,] "SP.DYN.LE00.IN"    "Life expectancy at birth, total (years)" 
## [3,] "SP.DYN.LE00.FE.IN" "Life expectancy at birth, female (years)"

Let’s get both male and female data by using the following codes:

  1. SP.DYN.LE00.MA.IN
  2. SP.DYN.LE00.FE.IN
male   <- WDI(country = "all", indicator = "SP.DYN.LE00.MA.IN", start = 1960, end = 2016)
female <- WDI(country = "all", indicator = "SP.DYN.LE00.FE.IN", start = 1960, end = 2016)

## Add gender column
male$gender   <- "Male"
female$gender <- "Female"

## Change column name
names(male)[3]   <- "life.expectancy"
names(female)[3] <- "life.expectancy"

## Combine data together
data <- rbind(male, female)

1. Bar Chart

The humble bar chart.

This is probably the most basic chart that exists in the work place. It makes sense to start here, with the objective of the graph to show that females have a higher life expectancy than males.

Using the default ggplot aesthetic, it’s clear that it’s not publish-worthy.

data %>%
  group_by(gender) %>%
  summarise(avg.life.expectancy = mean(life.expectancy, na.rm = T)) %>%
  ggplot(aes(x = gender, y = avg.life.expectancy)) +
  geom_col() +
  labs(title = "Life Expectancy",
       subtitle = "Worldwide")

The first step in reducing the non-data ink is to subtract unnecessary non-data ink. Let’s first identify the non-data ink in the graph below:

  • Grey background
  • Vertical lines
  • Horizontal lines
  • X-axis title (gender)

Removing these elements from the graph does not lose any meaning. So let’s do that first.

data %>%
  group_by(gender) %>%
  summarise(avg.life.expectancy = mean(life.expectancy, na.rm = T)) %>%
  ggplot(aes(x = gender, y = avg.life.expectancy)) +
  geom_col() +
  theme_minimal() +
  theme(panel.grid = element_blank(),
        axis.ticks.y = element_line()) +
  labs(x = "",
       title = "Life Expectancy",
       subtitle = "Worldwide")

The next step is to de-emphasise and regularise the remaining non-data ink. In this case, the non-data ink consists of the y-axis title, because it’s important that the reader knows what the y-axis represents, but it could be more succinct (given the title).

Personally, I’d like to position it near the top of the chart, so I’ll change the title to “Age”, and change the location.

data %>%
  group_by(gender) %>%
  summarise(avg.life.expectancy = mean(life.expectancy, na.rm = T)) %>%
  ggplot(aes(x = gender, y = avg.life.expectancy)) +
  geom_col() +
  theme_minimal() +
  theme(panel.grid = element_blank(),
        axis.ticks.y = element_line(),
        axis.title.y = element_text(margin = margin(t = 0, r = 10, b = 0, l = 0),
                                    hjust = 0.96)) +
  labs(x = "", y = "Age",
       title = "Life Expectancy",
       subtitle = "Worldwide")

The next step is to enhance the data ink.

In this chart, the data ink is the ink in the bar charts. This is the element that represents the data.

This includes the elements of:

  • Width
  • Orientation
  • Size
  • Colour
  • Hue

I’m quite happy with most of the elements, except for colour.

It currently lacks “life”, which I think is important given its about life expectancy. So I’ll change the colour palette to reflect this.

data %>%
  group_by(gender) %>%
  summarise(avg.life.expectancy = mean(life.expectancy, na.rm = T)) %>%
  ggplot(aes(x = gender, y = avg.life.expectancy, fill = gender)) +
  geom_col(show.legend = FALSE ) +
  scale_fill_manual(values = c("#88BA7B", "#88BA7B")) +
  theme_minimal() +
  theme(panel.grid = element_blank(),
        axis.title.y = element_text(margin = margin(t = 0, r = 10, b = 0, l = 0),
                                    hjust = 0.96),
        # axis.line.x = element_line(colour = "grey50"),
        axis.ticks.y = element_line()) +
  labs(x = "", y = "Age",
       title = "Life Expectancy",
       subtitle = "Worldwide")

Finally, I think there’s a little too much white space for this graph, so I’d like to add a few more elements, as a matter of personal preference:

  • Add tick marks on the y-axis to increment by 10
  • Increase the y-axis limit to 70, as it currently goes beyond 60, and to create more white space in the top-right corner
  • Add a horizontal line on the x-axis to create some balance between the x and y-axis

The final result is this.

data %>%
  group_by(gender) %>%
  summarise(avg.life.expectancy = mean(life.expectancy, na.rm = T)) %>%
  ggplot(aes(x = gender, y = avg.life.expectancy, fill = gender)) +
  geom_col(show.legend = FALSE ) +
  scale_fill_manual(values = c("#88BA7B", "#88BA7B")) +
  theme_minimal() +
  theme(panel.grid = element_blank(),
        axis.title.y = element_text(margin = margin(t = 0, r = 10, b = 0, l = 0),
                                    hjust = 0.96),
        axis.line.x = element_line(colour = "grey50"),
        axis.ticks.y = element_line()) +
  scale_y_continuous(breaks = seq(0, 90, by = 10),
                     limits = c(0, 70)) +
  labs(x = "", y = "Age",
       title = "Life Expectancy",
       subtitle = "Worldwide")

2. Line chart

Line charts are also one of the most basic charts to use, and should mainly be used to represent time-series data.

I’ve seen plenty of examples where line charts have been used for categorical data; don’t do that….

This time, my objective is to see how life expectancy at birth has changed over time.

We start with a standard out-of-the-box ggplot, which looks like this:

ggplot(data = data[data$country == "Hong Kong SAR, China", ],
       aes(x = year, y = life.expectancy, colour = gender)) +
  geom_line() +
  labs(title = "Life Expectancy",
       subtitle = "Hong Kong SAR, China")

In terms of reducing the non-data ink and subtract unnecessary non-data ink, I’ll remove the:

  • Grey background
  • Vertical lines
  • x-axis title

In terms of de-emphasising and regularise the remaining non-data ink, I’ll:

  • Change y-axis label
  • Shift y-axis label upwards
  • Move the legend to the bottom right inside the chart area

In terms of enhancing the data ink, I’ll

  • Increase the thickness of each line
  • Change the colour of each line

And finally, I’ll add a few elements that I had before to balance out the chart:

  • Add more tick marks on the x and y axes
  • Add a horizontal bar on the x-axis

Note: The y-axis does not start on zero, unlike bar charts, because in the case of life expectancy, there is no country where life expectancy is zero. Also, the objective of this graph is to see the change over time, and starting the y-axis on 0 would not necessarily achieve this.

The final result is shown below:

ggplot(data = data[data$country == "Hong Kong SAR, China", ],
       aes(x = year, y = life.expectancy, colour = gender)) +
  geom_line(size = 1) +
  scale_colour_manual(values = c("#F18C8E", "#568EA6")) +
  theme_minimal() +
  theme(panel.grid.major.x = element_blank(),
        panel.grid.minor.x = element_blank(),
        panel.grid.minor.y = element_blank(),
        panel.grid.major.y = element_line(linetype = "dotted", colour = "grey80"),
        axis.title.y = element_text(margin = margin(t = 0, r = 10, b = 0, l = 0),
                                    hjust = 0.96),
        axis.line.x = element_line(colour = "grey50"),
        axis.ticks.y = element_line(),
        axis.ticks.x = element_line(),
        legend.position = c(0.9, 0.2)) +
  scale_y_continuous(breaks = seq(0, 100, by = 5), 
                     limits = c(60, 90)) +
  scale_x_continuous(breaks = seq(1960, 2020, by = 10)) +
  labs(x = "", y = "Age",
       title = "Life Expectancy",
       subtitle = "Hong Kong SAR, China")

Scatter plots

The last of the basic plots that I want to touch on are scatter plots.

These are good to show the relationship between two variables.

In the example below, I want to show the gender-bias in life expectancy between females and males, as females often have a longer life than males. I’ll also add emphasis on two places: Australia and Hong Kong (I live in Australia, but my family is from Hong Kong).

Again, the out-of-the-box ggplot looks like this (it’s getting dull and boring):

data %>%
  filter(year == 2016) %>%
  select(-iso2c) %>%
  spread(gender, life.expectancy) %>%
  ggplot(aes(x = Male, y = Female)) +
  geom_point() +
  labs(title = "Gender gap in Life Expectancy")

I’ve applied most of the same processes as the previous examples to the graph below.

Some differences are:

  • Axis labels are inside the axes, to use some available white space, and increase the chart size.
  • Omit lines for each axis, as this subtracts from the emphasis I’m placing on Australia and Hong Kong.
  • Greyed out the points for most of the data, and added colour only for the two places of interest.
  • Removed the legend, as it’s clear that each point represents a country/place.
  • Add a diagonal line to represent “equity”, to show the bias.

The result is shown below:

## Extract Australia
aus <- data %>%
  filter(year == 2016,
         country == "Australia") %>%
  select(-iso2c) %>%
  spread(gender, life.expectancy)

## Extract Hong Kong
hk <- data %>%
  filter(year == 2016,
         country == "Hong Kong SAR, China") %>%
  select(-iso2c) %>%
  spread(gender, life.expectancy)

size <- 3
shape <- 21

data %>%
  filter(year == 2016) %>%
  select(-iso2c) %>%
  spread(gender, life.expectancy) %>%
  ggplot(aes(x = Male, y = Female)) +
  geom_point(fill = "grey80", 
             colour = "grey80",
             size = size,
             shape = shape,
             alpha = 0.3) +
  geom_abline(colour = "grey70", linetype = "dotted", size = 1) +
  geom_point(data = aus, 
             aes(x = Male, y = Female),
             fill = "#F58426", 
             colour = "#F58426",
             shape = shape,
             size = size) +
  annotate(geom = "text", label = "Australia", x = 85, y = 82, 
           size = 3.5, colour = "#F58426") +
  annotate(geom = "segment", x = 83, y = 82, xend = 81, yend = 84, 
           size = 0.5, colour = "#F58426") +
  geom_point(data = hk, 
             aes(x = Male, y = Female),
             fill = "#007F91", 
             colour = "#007F91",
             shape = shape,
             size = size) +
  annotate(geom = "text", label = "Hong Kong", x = 77, y = 89,

    annotate(geom = "segment", x = 79.5, y = 88.5, xend = 80.5, yend = 87.5,

  theme_minimal() +
  theme(panel.grid = element_blank(),
        axis.ticks.y = element_line(),
        axis.ticks.x = element_line()) +
  scale_x_continuous(limits = c(50, 90)) +
  scale_y_continuous(limits = c(50, 90)) +
  labs(title = "Gender gap in Life Expectancy",
       subtitle = "Australia and Hong Kong",
       x = "", y = "") +    # replace axis labels  with annotations inside the chart
  annotate(geom = "text",
           label = "Male", 
           fontface = 2,
           x = 80, y = 50, colour = "grey50") +
  annotate(geom = "text",
           label = "life expectancy", 
           x = 85, y = 50, colour = "grey50") +
  annotate(geom = "text",
           label = "Female", 
           fontface = 2,
           x = 50, y = 75, colour = "grey50",
           angle = 90) +
  annotate(geom = "text",
           label = "life expectancy", 
           x = 50, y = 84, colour = "grey50",
           angle = 90)

A note on Colours

Colours contribute a lot to the look and feel of a visualisation. When plots use standard colours, it’s easy to tell how much effort has been put into producing a data viz.

I also have weakness in differentiating reds and greens (red-green colour blindness affects up to 8% of males), and so I avoid using colour palettes that contain these colours.

I like to go to Adobe Colors to search for a colour palette to use.
